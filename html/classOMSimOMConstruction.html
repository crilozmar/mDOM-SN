)

The physical volume of the constructed module is produced here. Also the harness is placed here. In the case of the mDOM, the harness is still not in the code, but its construction/placement should probably be called from here!

Parameters
pPositionG4ThreeVector with position of the module (as in G4PVPlacement())
pRotationG4RotationMatrix with rotation of the module (as in G4PVPlacement())
pMotherG4LogicalVolume where the module is going to be placed (as in G4PVPlacement())
pIncludeHarnessbool Harness is placed if true
pNameExtensionG4String name of the physical volume. You should not have two physicals with the same name
